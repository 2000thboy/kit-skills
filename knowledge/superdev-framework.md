# Super Dev Framework

## 一句话

Super Dev 是 AI 编码交付治理层。它适合前后端产品交付，强调 research、PRD、architecture、UI/UX、spec、实现、质量门禁、proof pack 和发布准备。

## 适合什么

- 前端/后端/全栈产品。
- UI 质量、运行时证明、发布准备很重要。
- 用户希望 AI 不是只生成代码，而是按阶段交付。
- 项目需要 research、三文档审批、质量门禁、交付包。

## 不适合什么

- 只是一个 skill。
- 只是一个小型 workflow。
- 只是 CLI harness。
- 用户还没决定产品对象，直接上 Super Dev 会显得很重。

## 和 KIT 的关系

KIT 负责先问清楚用户要开发什么。若用户要做前后端产品，尤其是 UI/UX 和发布交付都重要，KIT 可以推荐 Super Dev 作为交付治理框架。

KIT 不复制 Super Dev 的流程。KIT 只在 PRD/SPEC/CHECKLIST 里记录是否 routed 到 Super Dev、谁负责、证据路径和 stop gate。

## 决策提示

- 开发效果：更像产品交付流程，不是聊天式写代码。
- 成本：阶段和文档更多，轻量项目会嫌重。
- 可反悔点：早期可以从 KIT 切到 Super Dev。
- 难反悔点：前端优先、UI gate、交付包结构一旦成为事实源，中途换回轻量 workflow 需要归档迁移。
