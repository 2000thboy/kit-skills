# User Testing — kit-skills

This folder is only for real user testing.

## What To Give A Tester

1. A GitHub repo link or release/package pointer.
2. A short task guide in `.test/user/guides/`.
3. A pass/fail checklist in `.test/user/acceptance/`.
4. A place for returned notes in `.test/user/feedback/`.
5. Returned screenshots or files in `.test/user/evidence/`.

## What Not To Put Here

- AI-simulated user feedback.
- Automated browser or CLI logs.
- npm pack dry-run output.
- Model-generated review reports.

Those belong under `.test/ai/`.
