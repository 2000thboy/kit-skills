# WorkBuddy Workflow Preset

## Progress Source

Use:

- `.plan/CHECKLIST.md` for task status.
- `.kit/config.json` for stage, blockers, owner, and next tasks.
- `.workflow/status.md` for workflow notes.

## Report Format

- Current stage
- Completed work
- Active blockers
- Next action
- Evidence path

## Rule

Do not create a parallel status system. Update `.plan/CHECKLIST.md` or `.kit/config.json` when status changes.
