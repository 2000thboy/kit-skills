## mineru-pdf-parser (Unpatched ❌)

### Vulnerabilities Found

| ID | Vulnerability | Evidence | Severity |
|----|--------------|----------|----------|
| P1 | **Path Traversal** | `mineru -p input.pdf` accepts arbitrary paths | 🔴 P0 |
| P2 | **Command Injection** | PDF filename passed to shell command | 🔴 P0 |
| P3 | **Resource Exhaustion** | No file size limit for PDF processing | 🟠 P1 |
| P4 | **Output Overwrite** | `-o output_dir` silently overwrites | 🟡 P2 |
| P5 | **XXE Risk** | PDF may contain XFA/XML forms | 🟡 P2 |

### Recommended Fixes
```python
# P1: Path validation
def validate_pdf_path(path: str) -> Path:
    p = Path(path).resolve()
    if ".." in str(p):
        raise ValueError("Path traversal detected")
    return p

# P2: Use list args instead of string
cmd = ["mineru", "-p", str(pdf_path), "-o", str(output_dir)]
subprocess.run(cmd)  # Safe

# P3: Size limit
MAX_PDF_SIZE = 500 * 1024 * 1024  # 500MB

# P4: Output protection
if output_dir.exists() and any(output_dir.iterdir()):
    raise ValueError("Output directory not empty")
```

## opencli-browser (Unpatched ❌)

### Vulnerabilities Found

| ID | Vulnerability | Evidence | Severity |
|----|--------------|----------|----------|
| B1 | **SSRF** | `web read --url <url>` accesses arbitrary URLs | 🔴 P0 |
| B2 | **Cookie/Session Leak** | Browser automation exposes all cookies | 🟠 P1 |
| B3 | **XSS via Eval** | `browser eval` returns untrusted page JS | 🟠 P1 |
| B4 | **Form Submission Bypass** | Users may bypass "read-only" eval restriction | 🟡 P2 |

### Notes
- SKILL.md explicitly marks eval as "read-only" and warns against form submission
- But `web read` has no URL whitelist
- No cookie isolation mechanism documented

### Recommended Fixes
```bash
# B1: URL whitelist
opencli web read --url <url> --allow-domains "example.com,api.example.com"

# B2: Cookie isolation
opencli browser --profile "analysis-profile" --no-cookies

# B3: Eval sandbox
opencli browser eval --sandbox --no-dom-access "..."
```
